Phase 01 · Foundations

Python + LLM Fundamentals

2–3 weeks

Python for senior Java engineers

  • Python 3.12/3.13 fast-track — type hints, dataclasses, async/await, decorators, mapped to Java equivalents
  • Modern tooling: uv package manager, ruff, pytest
  • Pydantic v2 — validation & schemas, the backbone of structured LLM outputs
  • FastAPI 0.11x — async REST services to wrap every agent you build

LLM fundamentals

  • Transformer intuition: tokens, context windows, temperature, why LLMs hallucinate
  • 2026 model landscape: GPT-5.x, Claude Opus 4.8 / Sonnet 4.6, Gemini 2.5/3, open-weight Llama 4 / Mistral / Qwen 3
  • Direct API calls with OpenAI SDK + Anthropic SDK — streaming, retries, token & cost accounting
  • Structured outputs — JSON mode + Pydantic schemas, the single most important production skill
  • Embeddings & cosine similarity; local experimentation with Ollama
Phase Assignment
Smart Ticket Classifier API
A FastAPI service that turns free-text support tickets into Pydantic-validated JSON (category, priority, sentiment, team) via the Anthropic/OpenAI SDK — with streaming, exponential-backoff retries, per-request cost logging, and a Docker image.
covers → Python · FastAPI · Pydantic · LLM APIs · structured outputs · production hygiene

Phase 02 · Reasoning

Prompting & Agentic Reasoning Patterns

2 weeks

Production prompt engineering

  • System vs. user prompts, few-shot examples, XML delimiters, output constraints
  • Prompts as versioned, tested code — stored in git
  • Context engineering — the 2026 evolution: what enters the window, what gets retrieved, what gets summarized

Reasoning patterns — know when to use each

  • CoT — step-by-step reasoning, and how native "thinking" models absorbed it
  • ReAct — the foundational loop: thought → action → observation → repeat
  • ToT — branching exploration; when it justifies the cost
  • Reflexion / self-critique, Plan-and-Execute, self-consistency voting
  • Tool / function calling — schemas, parallel calls; the bridge from chatbot to agent

What an agent actually is

  • Agent = LLM + tools + memory + loop + goal; Anthropic's workflow taxonomy (chaining, routing, orchestrator-workers, evaluator-optimizer)
  • When not to build an agent — a deterministic workflow is often correct (a strong interview answer)
Phase Assignment
ReAct Research Agent — from scratch, no framework
A pure-Python ReAct loop over the raw OpenAI/Anthropic SDK with three tools (calculator, Tavily web search, file writer), a max-iteration guard, and a Reflexion pass that critiques and retries once. Framework-free is the best interview prep there is.
covers → ReAct · CoT · Reflexion · tool calling · agent-loop internals
3
Phase 03 · Retrieval

RAG — Basic → Advanced → Graph → Agentic

3 weeks

Basic RAG

  • Load → chunk → embed → store → retrieve → generate; chunking strategies
  • Vector DBs: pgvector 0.8 on PostgreSQL (the enterprise default), Qdrant 1.x, Chroma, Pinecone
  • Build with LangChain 1.x first; know when LlamaIndex 0.14.x wins (complex ingestion)

Advanced RAG

  • Hybrid search — dense vectors + BM25, Reciprocal Rank Fusion
  • Re-ranking with Cohere Rerank 3.5 or open-source BGE-reranker
  • Query transformation: HyDE, multi-query expansion, routing; contextual retrieval
  • Access-control-aware retrieval — critical in fintech, your niche
  • RAG evaluation with RAGAS 0.3.x — faithfulness, relevancy, context precision/recall

Graph RAG

  • Where vector RAG fails: multi-hop, relationships, "global" questions
  • Build with Neo4j 5.x + neo4j-graphrag for enterprise knowledge graphs; Microsoft GraphRAG 2.x for corpus-wide summarization

Agentic RAG

  • Retrieval becomes an agent: decide, retrieve, grade, rewrite, loop until confident
  • Patterns: Corrective RAG, Self-RAG, router-based multi-index — built on LangGraph 1.2.x, the standard stack in postings
Phase Assignment
Regulatory Knowledge Assistant (Agentic RAG)
Ingest PSD2/PCI-DSS PDFs into LangChain + pgvector with hybrid BM25 search and Cohere re-ranking, then wrap it in a LangGraph loop that grades chunks, rewrites weak queries, and falls back to web search. Add a small Neo4j entity graph for multi-hop questions and report RAGAS faithfulness before/after re-ranking.
covers → basic · hybrid · re-ranking · Graph RAG · Agentic RAG · RAG evals

Phase 04 · Orchestration

Agent Frameworks & Multi-Agent Systems

3 weeks

LangGraph 1.2.x — the primary framework

  • StateGraph: typed state, nodes, conditional edges, cycles; create_agent() + middleware
  • Durable execution — Postgres/SQLite checkpointers; maps to your saga experience
  • Human-in-the-loop — interrupts, approval gates, time-travel replay
  • Memory: thread state vs. long-term store; multi-agent shapes — supervisor, hierarchical teams, swarm/handoffs

Vendor agent SDKs — know three, go deep on one

  • OpenAI Agents SDK 0.1x — agents, handoffs, three-tier guardrails, sandboxed execution, built-in tracing
  • Claude Agent SDK 0.1.x — "give the agent a computer": subagents, hooks, deepest MCP integration
  • Google ADK 2.0 — multi-language (Python/Java/Go/TS), graph workflows, native A2A — its Java support is your differentiator
  • CrewAI 1.x role-based crews (conceptual); Pydantic AI 1.x (awareness)

Cross-framework design patterns

  • Orchestrator-workers, evaluator-optimizer, router, fan-out/fan-in
  • Saga-style compensation for side-effecting agent actions; context isolation for sub-agents
Phase Assignment
Multi-Agent Fraud Triage System
A LangGraph supervisor over three specialists — transaction analysis, customer-history RAG (Phase 3 stack), and report writer — with a human-in-the-loop interrupt before any account block and Postgres checkpointing so the flow survives restarts. Rebuild a 2-agent version in the OpenAI Agents SDK and write a one-page framework comparison.
covers → LangGraph state/HITL/durability · supervisor pattern · vendor SDK · trade-off reasoning

Phase 05 · Protocols

MCP & A2A — The Agent Protocol Layer

2 weeks

Model Context Protocol spec 2025-11-25

  • Hosts, clients, servers; primitives — tools, resources, prompts; sampling & elicitation
  • Transports: stdio vs. Streamable HTTP; stateless vs. stateful sessions; OAuth 2.1 for remote servers
  • Build in Python with MCP SDK 1.x / FastMCP 2.x; in Java with Spring AI MCP Spring AI 1.1.x — your Java superpower applies directly
  • Consume MCP from Claude Agent SDK (native), LangGraph via langchain-mcp-adapters, OpenAI Agents SDK
  • MCP security: tool poisoning, prompt injection via descriptions, confused-deputy, allowlisting

Agent-to-Agent Protocol (Linux Foundation)

  • Agent Cards, tasks, artifacts, long-running lifecycle; ACP merged into A2A in 2026
  • MCP = agent↔tool; A2A = agent↔agent. Build with a2a-python, native in Google ADK & CrewAI
Phase Assignment
Enterprise MCP Gateway
Two MCP servers: a FastMCP Python server exposing your Phase 3 RAG as tools, and a Spring AI MCP Java server wrapping a mock core-banking API over OAuth-protected Streamable HTTP. Connect both to Claude Code and to the LangGraph agent. Bonus: publish the fraud-triage supervisor as an A2A agent with an Agent Card.
covers → MCP in Python & Java · transports · OAuth · multi-framework clients · A2A

Phase 06 · Cloud

Cloud-Based Agentic AI Platforms

3 weeks

AWS — primary (matches your SA-Pro)

  • Amazon Bedrock: Converse API, Knowledge Bases (managed RAG), Bedrock Guardrails
  • Bedrock AgentCore (GA): Runtime (session-isolated microVMs), Gateway (APIs → MCP tools), Memory, Identity, Code Interpreter, Browser, Observability
  • Framework-agnostic hosting — deploy LangGraph, CrewAI, or Strands agents; host MCP servers
  • Strands Agents SDK 1.x — AWS's open-source model-driven framework, the idiomatic AgentCore pairing
  • Senior-level depth: VPC/private connectivity, IAM for agents, cost controls

Azure / Microsoft — highest enterprise demand in Europe

  • Microsoft Foundry + Foundry Agent Service — managed agents, Azure AI Search, MCP endpoints, Foundry Toolbox
  • Microsoft Agent Framework 1.0 GA — the AutoGen + Semantic Kernel merger; Python & .NET, OTel built in
  • Agent 365 / Entra Agent ID governance — a Director-level talking point

Google Cloud — awareness

  • Vertex AI Agent Engine + ADK 2.0 deployment; one conceptual walkthrough

Phase 07 · AgentOps

Evaluation, Observability, Security & Production

2–3 weeks

Evaluation — the #1 skill gap employers cite

  • Prompt unit tests, LLM-as-judge, regression suites, A/B testing
  • Agent-specific: trajectory evaluation (right tools, right order?), task-completion rate, cost/latency budgets
  • Tools: LangSmith, RAGAS 0.3.x, DeepEval / promptfoo; eval-gated deployment in GitHub Actions

Observability

  • Spans for LLM calls, tools, retrievals; token & cost attribution per session
  • OpenTelemetry GenAI semantic conventions + LangSmith or Langfuse 3.x (self-hostable, popular in EU/regulated firms) or Arize Phoenix

Security & guardrails

  • OWASP LLM Top 10: prompt injection, insecure output handling, excessive agency, exfiltration
  • Defense-in-depth: NeMo Guardrails, Guardrails AI, Bedrock/OpenAI guardrails; least-privilege tools; HITL for irreversible actions
  • EU AI Act awareness — relevant to your Amsterdam/London/Berlin targets

Production engineering for agents

  • Kubernetes or serverless deployment; SSE/WebSocket streaming
  • Timeouts, circuit breakers, idempotent tools, saga compensation, semantic caching
  • Versioned prompts, models, and eval datasets; canary rollouts of model upgrades
Phase Assignment
Harden and Ship
A LangSmith dataset of 30 cases — including 5 prompt-injection attacks and 5 tool failures — with trajectory evals gating GitHub Actions, OTel GenAI traces into Langfuse, NeMo Guardrails input rails, and a dashboard for cost-per-triage, p95 latency, and eval trend. Publish "what broke, what I measured, what I fixed."
covers → evals · LLM-as-judge · CI/CD gating · OTel tracing · guardrails · security testing

Java-Native Track + Portfolio

Java/Spring agentic stack — your differentiator

  • LangGraph4j 1.8.x — LangGraph's graph model in Java

Capstone — pick one, build deep, publish with a demo video
